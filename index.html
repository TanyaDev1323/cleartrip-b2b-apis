Content APIs

Content APIs are designed to fetch hotel and room content data. These APIs are intended for periodic content updates and should not be used within the user flow.

Purpose:

  • Fetch and refresh static hotel and room content data.

Usage Recommendation:

  • Frequency: Refresh content weekly.
  • Timing: Perform updates during off-peak hours (1 AM to 8 AM IST).

Note:

  • Adhering to the recommended usage ensures minimal impact on system performance and user experience.

Get Location List

This Content API provides detailed location hierarchy and metadata for a given locationType.

Features:

  • Retrieves location details along with hierarchical relationships.
  • Supports efficient pagination for large datasets.
  • Provides associated metadata for each location.

Get Hotel List

This API retrieves a list of hotels for a specified location based on the provided location ID and location type.

Features:

  • Fetches hotel information for a specific location.
  • Supports pagination for efficient retrieval of large hotel lists.
  • Ensures scalable and optimized data access.

This allows users to seamlessly obtain hotel details for dynamic location-based searches.

Constraint:

  • locationType (String): Currently accepts ["CITY"] and ["LOCALITY"]. We expect to expand the supported types in the future.

Note: If hasNextPage is true in the response, a nextPageToken will be included. Pass this token as the nextPageToken query parameter in your next request to retrieve subsequent pages.

string

pageSize

string

Mandatory only for fetching subsequent pages.

required

Get Incremental Update Hotel List

This Incremental Update Content API allows partners to request a list of hotels that have been updated after a specified lastUpdatedAt timestamp.

Features:

  • Retrieves hotel IDs for hotels updated after the given timestamp.
  • Supports pagination through a nextPageToken for fetching results in chunks.
  • Enables efficient incremental synchronization of hotel content.

Designed to help partners maintain up-to-date hotel content with minimal data transfer.

Note: If hasNextPage is true in the response, a nextPageToken will be included. Pass this token as the nextPageToken query parameter in your next request to retrieve subsequent pages.

query Parameters
lastUpdatedAt
integer <int64>

Epoch timestamp for last updated time.

Get Hotel Profile

Features:

  • Fetches comprehensive hotel details.
  • Includes hotel policies, amenities, and other relevant information.
  • Supports accessing complete hotel profiles for display or further processing.

Designed to provide rich and detailed hotel content efficiently.

Constraint:

  • Maximum stay duration: 28 nights
  • Maximum age for children: 11 years (child ages must be 0–11 inclusive)
  • Limit: 6 rooms per request
  • Single room capacity: 1 to 8 adults

Get Hotel Profile in Batch

This API retrieves detailed profile information for a list of hotels based on the provided hotel IDs.

Features:

  • Fetches comprehensive details for multiple hotels.
  • Includes hotel policies, amenities, and other relevant information.
  • Supports batch retrieval of hotel profiles for efficient data access.

Ideal for scenarios where multiple hotel profiles need to be fetched in a single request.

Constraint:

  • Max allowed batch size for hotelIds is 50.
  • While the Hotel Batch Profile doesn’t currently support pagination via hasNextPage, the response times remain consistently fast, typically loading within a few seconds

Trip API

The Get Trip API allows clients to retrieve detailed trip information by querying either a Cleartrip Trip ID (tripId) or a Client Trip ID (affiliateTripReference).

Identification Parameters:

  • Cleartrip Trip ID (tripId): This ID is provided by Cleartrip in the response of the booking API. It is used to query and retrieve detailed information about the trip.

  • Client Trip ID (affiliateTripReference): This ID is generated by the client and included in the request to the booking API. It serves as an alternative to the Cleartrip Trip ID for retrieving trip details.

By using either of these parameters, clients can seamlessly access the required trip details.
